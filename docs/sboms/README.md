## Overview

This directory contains SPDX-format SBOMs for all container images. I generate these to provide transparency into all software components used in this project.

---

## Files

- `frontend-sbom.json` - Complete dependency list for React frontend
- `backend-sbom.json` - Complete dependency list for Node.js backend

---

## Purpose

I generate SBOMs to support:

- **Security vulnerability tracking** - Know exactly what's in the containers
- **License compliance auditing** - Verify all licenses are compatible
- **Supply chain risk management** - Track dependency origins
- **Regulatory compliance** - Meet NIST and SLSA requirements

---

## Generation

SBOMs are automatically generated by Syft during the CI/CD pipeline (Stage 2: Build & Container Security).

**Tool:** Syft by Anchore  
**Format:** SPDX 2.3 (JSON)  
**Frequency:** Every release

---

## Verification

These SBOMs are cryptographically signed and attached to container images.

**Verify SBOM signature:**

```bash
cosign verify-attestation --type spdx \
  --certificate-identity-regexp "https://github.com/AkingbadeOmosebi/brewsecops/.*" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  ghcr.io/akingbadeomosebi/brewsecops-frontend:latest \
  --output-file sbom.json
```

**View SBOM contents:**

```bash
cat sbom.json | jq -r '.payload' | base64 -d | jq
```

**Extract package list:**

```bash
cat sbom.json | jq -r '.payload' | base64 -d \
  | jq '.packages[] | {name: .name, version: .versionInfo, license: .licenseConcluded}'
```

---

## SBOM Contents

Each SBOM includes:

- **Package names** - All npm and OS packages
- **Versions** - Exact versions installed
- **Licenses** - MIT, Apache-2.0, GPL, etc.
- **Package URLs** - PURL format for unique identification
- **File locations** - Where packages are installed

## Use Cases

### Vulnerability Tracking

When a new CVE is announced, I can quickly check if affected packages are in the containers:

```bash
cat frontend-sbom.json | jq -r '.payload' | base64 -d \
  | jq '.packages[] | select(.name == "PACKAGE_NAME")'
```

### License Compliance

Verify all licenses are acceptable:

```bash
cat frontend-sbom.json | jq -r '.payload' | base64 -d \
  | jq '.packages[].licenseConcluded' | sort | uniq
```

### Dependency Audit

List all dependencies for security review:

```bash
cat backend-sbom.json | jq -r '.payload' | base64 -d \
  | jq '.packages[] | {name, version: .versionInfo}' | less
```

## Compliance Standards

These SBOMs follow:

- **SPDX 2.3** - Industry standard format
- **NIST Guidelines** - Executive Order 14028 requirements
- **SLSA Level 3** - Supply chain security framework
- **CycloneDX compatible** - Can be converted if needed
